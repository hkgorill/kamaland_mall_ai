# 프로젝트명: KAMALAND_MALL AI (AI 기반 무재고 위탁 판매 자동화 솔루션)
# 타겟 환경: Local PC (Python, Streamlit)
# AI 모델: Google Generative AI (Gemini 1.5 Pro 텍스트 및 Imagen 기반 이미지 생성)

## 1. 프로젝트 개요
사용자가 도매 상품을 소싱하고, 매력적인 마케팅 이미지와 후킹 문구를 생성하여 최종적으로 구매 전환율을 높이는 상세페이지를 렌더링하는 업무 자동화 웹 애플리케이션입니다.

## 2. 공통 UI/UX 요구사항
- **프레임워크:** `streamlit`을 사용하며 `st.set_page_config(layout="wide", page_title="KAMALAND_MALL AI")` 적용.
- **테마:** Dark Mode 스타일을 기본으로 구성.
- **사이드바 네비게이션:**
  - 제목: `✨ KAMALAND_MALL AI`
  - 메뉴 리스트 (Radio button): `[대시보드, 오늘 뭐 팔지? (소싱), 대표 이미지 만들기, 상세페이지 후킹 문구, 상세페이지 생성]`

## 3. 핵심 기능 명세

### 3.1. 오늘 뭐 팔지? (소싱)
- **목적:** 트렌디한 도매 상품 키워드 추출
- **UI 레이아웃:** 2단 Column 구조 (좌측: 제어부 1 / 우측: 결과부 1.5 비율)
- **입력 (Input):** 추출할 개수 (`st.number_input`, 1~20개 제한)
- **액션:** '🔄 키워드 추출하기' 버튼 (Primary 속성)
- **출력 (Output):** Gemini 1.5 Pro API를 호출하여 입력된 개수만큼의 소형/생필품 키워드를 글머리 기호(`-`) 형태로 반환받아 화면에 렌더링.

### 3.2. 대표 이미지 만들기
- **목적:** 원본 상품의 형태는 보존하되 고품질의 마케팅 컷 5장 생성
- **UI 레이아웃:** 2단 Column 구조
- **입력 (Input):** 원본 제품 사진 업로드 (`st.file_uploader`, jpg/png)
- **액션:** '✨ 이미지 5장 생성하기' 버튼 (사진 업로드 전까지 Disabled)
- **처리 로직:** Google AI Studio의 Imagen API를 호출하여 Inpainting 방식으로 상품 형태는 보존하고 배경/조명을 변경.
- **출력 (Output):** 각기 다른 5가지 조명/배경 컨셉(예: 미니멀, 고급 호텔 등)이 적용된 이미지 갤러리 렌더링.

### 3.3. 상세페이지 후킹 문구
- **목적:** 소비자의 구매 욕구를 자극하는 설득형 카피라이팅 도출
- **UI 레이아웃:** 단일 또는 2단 구조
- **입력 (Input):** 상품 정보 텍스트 입력창 (`st.text_area`) 또는 타겟 상품 URL 입력창 (`st.text_input`).
- **액션:** '✍️ 후킹 문구 추출하기' 버튼
- **처리 로직:** Gemini 1.5 Pro 모델에 "KAMALAND_MALL 마케팅 전문가 페르소나"를 부여하고 PAS(Problem-Agitation-Solution) 프레임워크 기반으로 프롬프트 전송.
- **출력 (Output):** 메인 타이틀, 서브 타이틀, 3가지 핵심 소구점 반환.

### 3.4. 상세페이지 생성 (통합 및 미리보기)
- **목적:** 추출된 이미지와 카피라이팅을 묶어 최종 업로드용 상세페이지 렌더링
- **UI 레이아웃:** 좌측(데이터 세팅) / 우측(실시간 미리보기 렌더링)
- **입력 (Input):** 앞서 생성된 이미지 및 텍스트 데이터 매핑.
- **액션:** '🚀 상세페이지 생성 및 미리보기' 버튼
- **처리 로직 (미리보기 렌더링):** HTML/CSS 템플릿을 활용하여 브라우저에서 실제 모바일 쇼핑몰을 보는 것처럼 수직 스크롤 형태의 상세페이지 조립. (`st.components.v1.html` 활용)
- **출력 (Output):** 시각적으로 완성된 상세페이지 화면 및 'HTML 코드 복사하기' 기능 제공.

## 4. 에러 핸들링 및 제약사항
- Google Generative AI API 연동 시 `try-except` 구문을 활용하여 API Key 오류 및 통신 실패 메시지를 노출 (`st.error`).
- 모든 액션 버튼 클릭 시 `st.spinner`를 사용하여 로딩 상태 처리.
